# Финтех приложение FinTrack

Приложение для учета личных финансов с аналитикой, бюджетами, целями, отчетами, учетом подписок и интеграцией с банковыми данными (в режиме симуляции).

## Назначение проекта

Проект выполнен в рамках производственной практики (кейс-задача №3) и развивает навыки проектирования и эксплуатации финтех-систем.

## Ключевые возможности

- Аутентификация и профили пользователей (Supabase Auth).
- Транзакции: создание, редактирование, удаление, расширенные фильтры.
- Категории: базовые и пользовательские.
- Бюджеты: лимиты, прогресс, предупреждения 80/100%, настраиваемые периоды.
- Аналитика: сводка, структура расходов, тренды, рекомендации.
- Отчеты: генерация и экспорт CSV/Excel/PDF.
- Финансовые цели: планирование и контроль прогресса.
- Подписки: регулярные списания, анализ переплат, действия со статусами.
- Интеграция с банком: подключение и импорт операций (симуляция).

## Технологический стек

### Frontend

- React 18
- TypeScript 5
- Vite
- MUI + Tailwind CSS
- Recharts
- react-hot-toast

### Backend / Data

- Supabase (PostgreSQL, Auth, Realtime, RLS)
- SQL миграции в `supabase/migrations`

### Тестирование

- Vitest
- Testing Library
- Cypress

## Безопасность

- RLS-политики для пользовательских таблиц.
- Разделение данных по `user_id`.
- Настраиваемые пороги и cadence уведомлений для контроля шумовых событий.

## Важные примечания

- Банковая интеграция реализована как симуляция.
- Подписки работают через `mock` provider по умолчанию; для реального API нужен набор env-параметров.
